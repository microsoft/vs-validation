# VSV0002 Add Requires.Range

Supported numeric parameters that are expected to be non-negative should usually be guarded
with `Requires.Range`.

## Example of a pattern that is flagged by this analyzer

The following code accepts a signed numeric parameter without a range guard.
This generates a diagnostic.

```cs
class Test
{
    void M(int count)
    {
        System.Console.WriteLine(count);
    }
}
```

## Solution

Add `Requires.Range` near the top of the member body.

```cs
using Microsoft;

class Test
{
    void M(int count)
    {
        Requires.Range(count >= 0, nameof(count));
        System.Console.WriteLine(count);
    }
}
```

The analyzer only applies to the supported signed numeric types recognized by the rule.
Unsigned numeric types are not flagged.
