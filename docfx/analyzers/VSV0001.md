# VSV0001 Add Requires.NotNull

Reference-type parameters in members with block bodies should usually be guarded with
`Requires.NotNull` at the top of the body.

## Example of a pattern that is flagged by this analyzer

The following code uses a reference-type parameter without guarding it first.
This generates a diagnostic.

```cs
class Test
{
    void M(string value)
    {
        System.Console.WriteLine(value);
    }
}
```

## Solution

Add `Requires.NotNull` near the top of the member body.

```cs
using Microsoft;

class Test
{
    void M(string value)
    {
        Requires.NotNull(value);
        System.Console.WriteLine(value);
    }
}
```

If a matching `Requires.NotNull` call already exists, no diagnostic is reported.
