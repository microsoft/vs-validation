# VSV0003 Use Requires.NotNull

Manual null-check-and-throw patterns can often be simplified to `Requires.NotNull`.

## Example of a pattern that is flagged by this analyzer

The following code performs a manual null check and throws `ArgumentNullException`.
This generates a diagnostic.

```cs
class Test
{
    void M(string value)
    {
        if (value is null)
        {
            throw new System.ArgumentNullException(nameof(value));
        }

        System.Console.WriteLine(value);
    }
}
```

## Solution

Replace the manual null check with `Requires.NotNull`.

```cs
using Microsoft;

class Test
{
    void M(string value)
    {
        Requires.NotNull(value);
        System.Console.WriteLine(value);
    }
}
```

The analyzer intentionally stays narrow: patterns with `else` blocks or different exception
types are not reported.
