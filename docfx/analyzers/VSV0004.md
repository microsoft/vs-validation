# VSV0004 Remove redundant Requires.NotNull argument name

`Requires.NotNull` already captures the first argument expression as the parameter name.

## Example of a pattern that is flagged by this analyzer

The following code repeats the parameter name explicitly.
This generates a diagnostic.

```cs
using Microsoft;

class Test
{
    void M(string value)
    {
        Requires.NotNull(value, nameof(value));
    }
}
```

## Solution

Remove the redundant second argument.

```cs
using Microsoft;

class Test
{
    void M(string value)
    {
        Requires.NotNull(value);
    }
}
```

The analyzer only reports cases where removing the argument preserves the captured
caller-expression text.
